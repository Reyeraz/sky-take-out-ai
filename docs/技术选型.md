- 推荐业务的实现
	- 使用 Redis 做热点对话的实现, Pgsql 做对话的持久化
	- 仅展示最新消息, 但是完整向 AI 展示当天的上下文
	- 最新消息可在 Redis 中持续一天, 否则当推荐业务查询过多时可能会卡数据库
- 客服业务的实现, 以下都可以使用 Tool 实现
	- 用户说自然语言，AI 直接解析：
	- “我昨天晚上八点的麻辣香锅怎么还没到”
	- “帮我取消今天的订单”
	AI 自动解析：
	- 用户 ID
	- 订单时间范围
	- 订单状态意图
	- 问题类型
	    然后**自动调后端口查询 / 操作**，不用人工拼参数。
